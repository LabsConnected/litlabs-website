### H-2. Autonomic Loop runtime verification (active task)

**File:** `tasks/active.json`
**Status:** `in_progress`
**Milestone:** "Autonomic Loop Wiring"

The 4 GET endpoints the banner pings (`/api/director/plan`, `/api/agents`, `/api/memory`, `/api/agent-tasks`) are confirmed 200-safe with graceful degradation (per `docs/chat-api-wiring-map.md` §5). The **GET probe is not the full loop**, though:

**Action:**
1. Add a `pnpm daemon:dev` script (e.g., `"daemon:dev": "tsx src/daemon.ts"`).
2. Run a smoke test: sign in → POST a director plan → POST a task → wait for the daemon to claim and execute it.
3. Either start the daemon in Railway/Render alongside the web, or document that the loop's execution is intentionally out-of-band and the banner is the only health surface.
