## 2. `src/lib/` Audit (58 files, ~7,500 lines)

### 2.1 The "Brain" File: `litt.ts` **[FIRST-PASS STALE]**

> ⚠ **Correction from the second pass:** `litt.ts` is **NOT the "AI brain"**. It is a notification dispatcher (Discord / webhook / push / email) that uses Supabase to persist notifications.

`src/lib/litt.ts` exports **`jarvis`** as a singleton:

```typescript
class LiTT {
  private config: NotificationConfig;
  async notify(payload: NotificationPayload): Promise<boolean> { ... }
  async sale(...)      // → Discord notification
  async signup(...)    // → Discord notification
  async agentCreated(...)
  async systemAlert(...)
  async cliEvent(...)
}
export const jarvis = new LiTT();
```

The file contains **zero** prompt building, **zero** LLM calls, **zero** agent orchestration. It only sends notifications.

**Where the actual "brain" lives** (`.clinerules` is misleading on this point):

| File                           | What it does                                                                                                                                                                                                       |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `src/lib/litt-context.ts`      | `buildJarvisPrompt`, `parseJarvisActions`, `collectJarvisContext` — the closest thing to a brain. **The `LiTTAssistantPanel` component imports `JarvisContext`, `JarvisAction`, `JarvisThinkResponse` from here.** |
| `src/lib/agentCommands.ts`     | `getCommandsForAgent`, `executeCommand` — the agent command system                                                                                                                                                 |
| `src/lib/AgentOrchestrator.ts` | `AgentOrchestrator` class — orchestration matrix                                                                                                                                                                   |
| `src/lib/agents.ts`            | Second `AgentOrchestrator` + `simulateAgentResponse`, `startBackgroundConversation`                                                                                                                                |
| `src/lib/director-graph.ts`    | `DirectorGraphPlanner.buildPlan(goal)` — director agent graph                                                                                                                                                      |
| `src/lib/llm.ts`               | `generateText`, `generateJSON`, `streamText`, `llmHealth` — LLM provider                                                                                                                                           |

The `litt.ts` file is exported as `jarvis` (line 331: `export const jarvis = new LiTT();`). This naming — file is `litt.ts`, export is `jarvis` — is the source of the documentation confusion.

**.clinerules** says (line 15): `src/lib/litt.ts — main LiT brain / assistant logic (NOT jarvis.ts)`. **This is incorrect.** `litt.ts` is notifications, not the brain.

**Action:** Update `.clinerules` and `.cursorrules` to point to `litt-context.ts` (or the cluster of files above) as "the brain", and clarify that `litt.ts` is the notification dispatcher. The `litt`/`jarvis` naming asymmetry in the file/export is intentional and is now confirmed as harmless.
